Apply for sudo privileges on a Netsoc server

We have decided to open up one of our servers to the public. "But wait, Netsoc already do that".

While we give free server space and web-hosting to all UCC students, we don't allow students to have sudo privileges on the server. If they did, they could break the whole server in an instant.

Enter feynman

We are giving sudo priviliges to all those who wish it (and we deem responsible enough to have it).

  • Things to remember:

  • Other people using the server have sudo as well. They might break stuff; you might break stuff.

  • You are trusted with sudo so that you can install stuff, run services and run commands that need to be run as root.

  • You are not being given sudo so that that you can restart or shutdown the server, uninstall applications that others might be using, lock us (the Netsoc SysAdmin Team) out of the server or anything else malicious that will effect the experience of others.

    (Even if you lock us out, we still have physical access to the server if needs be).

Any breaches of trust will be acted on immediately.

Give us your email address to contact you on and we will get in touch with you. We just want to get a feel for you and your intentions with the server. Try to have an idea of what you would set up or do with the server when you approach us. We just have to protect our own :)

Once we feel that you're all good, we will make an account for you and give you privileges. You will then be SSHing to feynman.netsoc.co
